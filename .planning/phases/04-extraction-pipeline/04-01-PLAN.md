    **File 1: `src/extraction/extraction.processor.ts`** (new file)

    Create the BullMQ WorkerHost. This is the first BullMQ processor in the codebase — no existing analog.

    ```typescript
    import { Processor, WorkerHost } from '@nestjs/bullmq';
    import { Logger } from '@nestjs/common';
    import type { Job } from 'bullmq';
    import { ExtractionService } from './extraction.service.js';
    import type { ExtractionJobPayload } from './extraction.types.js';

    @Processor('extraction', { concurrency: 3 })
    export class ExtractionProcessor extends WorkerHost {
      private readonly logger = new Logger(ExtractionProcessor.name);

      constructor(private readonly extractionService: ExtractionService) {
        super();
      }

      async process(job: Job<ExtractionJobPayload>): Promise<void> {
        const correlationId = job.id ?? 'unknown';
        this.logger.debug(
          `[${correlationId}] Processing job attempt=${job.attemptsMade} sourceType=${job.data.sourceType}`,
        );
        await this.extractionService.runGraph(job.data, correlationId);
        this.logger.debug(`[${correlationId}] Job complete`);
      }
    }
    ```

    Note: `runGraph` does not exist on ExtractionService yet — plan 04-04 adds it. The processor will compile because TypeScript checks are run at build time, and plan 04-04 is in the same phase. If you want the file to compile independently now, add a temporary method stub comment — but do NOT add the method to the service yet (plan 04-04 owns that file).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Redis connection string | REDIS_HOST + REDIS_PORT read from env; malicious env could point to attacker-controlled Redis |
| BullMQ job payload | Jobs enqueued by ExtractionService; processor trusts payload shape |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Information Disclosure | REDIS_HOST / REDIS_PORT env vars | mitigate | Never log the connection string; ConfigService.getOrThrow reads from process.env which is not serialized to logs |
| T-04-01-02 | Tampering | ExtractionJobPayload shape | mitigate | ExtractionProcessor receives typed Job<ExtractionJobPayload>; plan 04-02 validates payload fields before node execution |
| T-04-01-03 | Denial of Service | BullMQ queue flooding | accept | Queue runs with concurrency: 3; BullMQ has built-in rate limiting hooks; queue flooding is a v2 concern (rate limit per userId is out of scope for v1) |
</threat_model>
