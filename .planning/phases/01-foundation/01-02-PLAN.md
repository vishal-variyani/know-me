  </read_first>
  <action>
    Create docker-compose.yml at the project root with this exact content (per D-01 and D-02):

    ```yaml
    services:
      db:
        image: supabase/postgres:15.14.1.107
        environment:
          POSTGRES_PASSWORD: postgres
          POSTGRES_DB: postgres
        ports:
          - "5432:5432"
        volumes:
          - pgdata:/var/lib/postgresql/data
        command: postgres -c config_file=/etc/postgresql/postgresql.conf
        healthcheck:
          test: ["CMD-SHELL", "pg_isready -U postgres"]
          interval: 10s
          timeout: 5s
          retries: 5

      redis:
        image: redis:7.2-alpine
        ports:
          - "6379:6379"
        volumes:
          - redisdata:/data
        healthcheck:
          test: ["CMD", "redis-cli", "ping"]
          interval: 10s
          timeout: 3s
          retries: 5

    volumes:
      pgdata:
      redisdata:
    ```

    Constraints (per locked decisions):
    - D-01: Image MUST be supabase/postgres (not postgres:15 vanilla) — pgvector 0.8.0 is bundled
    - D-02: ONLY db and redis services — no Studio, GoTrue, or Realtime in this compose file
    - POSTGRES_PASSWORD: postgres is acceptable for local dev only
    - Both services MUST have healthcheck blocks — Plan 03 depends on the db being healthy before pushing migrations

    After writing the file, attempt to pull and start the services if Docker is available:
    ```bash
    docker compose up -d 2>&1 | tail -20
    ```

    If Docker is not available on the machine, document the manual verification step in the task output and proceed. The healthcheck verification is gated on Docker availability.
